A **Literal** is a fixed value assigned directly in source code.

- ✔ Represents constant data
- ✔ Written directly in code
- ✔ Compiler infers type automatically (mostly)

<Code lang="java" title="What a literal looks like" code={`int x      = 10;      // 10    → integer literal
double d   = 10.5;    // 10.5  → floating-point literal
boolean b  = true;    // true  → boolean literal`} />

---

## 🗺️ Classification

| Literal | Example | Notes |
|---------|---------|-------|
| Integral | `10`, `0x1F`, `0b101` | Default type: `int` |
| Floating Point | `10.5`, `1.5e3` | Default type: `double` |
| Boolean | `true`, `false` | Only two values |
| Character | `'A'`, `'\n'`, `'\u0041'` | Single quotes, 2 bytes |
| String | `"Hello"` | String Constant Pool |
| null | `null` | Object references only |

---

## 1️⃣ Integral Literals

Whole numbers. Default type is **`int`**.

### Number Systems

| System | Base | Prefix | Example | Value |
|--------|------|--------|---------|-------|
| Decimal | 10 | *(none)* | `10` | 10 |
| Octal | 8 | `0` | `010` | 8 |
| Hexadecimal | 16 | `0x` / `0X` | `0x10` | 16 |
| Binary | 2 | `0b` / `0B` | `0b1010` | 10 |

<Aside type="note">
Java is **not case-sensitive** for literal prefixes — `0xFF`, `0XFF`, `0xff` all work. This is one of the very few places where Java ignores case.
</Aside>

<Code lang="java" title="Number system examples" code={`int x = 10;    // Decimal → 10
int y = 010;   // Octal   → 8
int z = 0x10;  // Hex     → 16
int w = 0b10;  // Binary  → 2

System.out.println(x + "----" + y + "----" + z); // 10----8----16`} />

#### Validity Quiz
<Code lang="java" title="✅ Valid"  code={`
int x = 0777;       // Octal (digits 0–7 only)
int x = 0xFACE;     // Hex uppercase
int x = 0xbeef;     // Hex lowercase
int x = 0xabb2cd;   // Hex longer value
`}/>

<Code lang="java" title="❌ Invalid" 
code={`
int x = 0786;   // C.E: '8' not valid in octal (digits 0–7 only)
int x = 0xBeer; // C.E: 'r' not a hex digit → ';' expected
`} />

### Underscore in Literals <Badge text="Java 7+" variant="success" />

<Code lang="java" title="Underscore for readability" code={`int million    = 1_000_000;       // ✅ readable!
long phone     = 98_765_43210L;   // ✅
double amount  = 1_23_456.78;     // ✅

// ⚠️ Rules — where underscore is NOT allowed:
int a = _100;     // ❌ can't START with _
int b = 100_;     // ❌ can't END with _
int c = 0x_FF;    // ❌ can't appear right after 0x prefix
int d = 1_0_0;    // ✅ valid (but pointless)`} />

<Aside type="danger">
```java
long big = 9_000_000_000;    // ❌ treated as int first → overflow at parse time!
long big = 9_000_000_000L;   // ✅ L suffix required for values > int range
```
</Aside>

## 2️⃣ Floating Point Literals

Real numbers (decimals). Default type is **`double`**.

<Code lang="java" title="✅ Valid"
code={`
double d  = 10.5;     // default — double literal
double d2 = 10.5d;    // explicit 'd' (optional)
float  f  = 10.5f;    // 'f' suffix required for float
double big   = 1.5e10;  // scientific notation → 15000000000.0
double small = 1.5e-3;  // scientific notation → 0.0015
`} />

<Code lang="java" title="❌ Invalid"
code={`
float f = 10.5;   // ❌ C.E: possible loss of precision (double → float)
//    FIX: float f = 10.5f;
`} />

<Aside type="tip">
**Memory trick:** *"Float is Fragile — needs `f` support."*
</Aside>

<Aside type="danger">
**Never use `float`/`double` for money.**
```java
System.out.println(0.1 + 0.2); // 0.30000000000000004 😱
// Use BigDecimal for financial calculations instead.
```
</Aside>

## 3️⃣ Character Literals

Single character, enclosed in **single quotes `' '`**, stored as Unicode (2 bytes).

### Four Valid Forms

<Code lang="java" title="All valid char literal forms" code={`char c1 = 'A';        // 1. Direct char literal
char c2 = 97;         // 2. Integral (ASCII/Unicode value) → 'a'
char c3 = '\u0041';   // 3. Unicode escape → 'A'
char c4 = '\n';       // 4. Escape sequence → newline`} />

<Code lang="java" title="✅ Valid"
code={`
char ch  = 'A';
char ch2 = 65;          // integer literal → 'A'
char ch3 = '\u0041';    // Unicode → 'A'
char ch4 = '\ubeef';    // valid Unicode escape
char ch5 = '\u0061';    // → 'a'
char ch6 = 0xFace;      // hex literal within 0–65535 range
`} />

<Code lang="java" title="❌ Invalid"
code={`
char ch = "A";     // ❌ C.E: double quotes → String, not char
char ch = a;       // ❌ C.E: variable 'a'ot declared
char ch = 'ab';    // ❌ C.E: unclosed character literal (multiple chars)
char ch = '';      // ❌ C.E: empty char not allowed
char ch = 65536;   // ❌ C.E: out of range (max 65535)
char ch = \u0062;  // ❌ C.E: missing quotes around unicode
char ch = '\iface';// ❌ C.E: illegal escape character (\i is not valid)
char ch = '\l';    // ❌ C.E: illegal escape character
`} />

### Escape Sequences

| Escape | Meaning |
|--------|---------|
| `\n` | New line |
| `\t` | Horizontal tab |
| `\r` | Carriage return |
| `\f` | Form feed |
| `\b` | Backspace |
| `\\` | Backslash |
| `\'` | Single quote |
| `\"` | Double quote |
| `\0` | Null character |

<Aside type="note">
Every **escape character** in Java acts as a **char literal** — it represents exactly one character.
</Aside>

## 4️⃣ String Literals

Sequence of characters, enclosed in **double quotes `" "`**.

- Stored in the **String Constant Pool** (heap)
- **Immutable** — once created, value never changes
- `String` is **not a primitive** — it's a class in `java.lang`

<Code lang="java" title="String basics" code={`String s = "Java";

// Concatenation
String result = "Hello" + "World";  // "HelloWorld"

// Number + String = String!
String mixed  = 10 + "20";          // "1020"  (NOT 30)
String mixed2 = "10" + 20;          // "1020"`} />

### Text Blocks <Badge text="Java 15+" variant="caution" />

<Code lang="java" title="Text blocks — no more \\n and + mess" code={`String json = """
        {
            "name": "Alice",
            "age": 25
        }
        """;

String html = """
        <html>
            <body>Hello</body>
        </html>
        """;`} />

## 5️⃣ Boolean Literals

Only **two values** allowed: `true` or `false`.

<Code lang="java" title="Boolean literals" code={`boolean flag = true;
boolean done = false;

// ✅ Clean idiom
if (isActive)  { }   // not: if (isActive == true)
if (!isEmpty)  { }

// ❌ NOT like C/C++
if (1)    { }   // C.E: int ≠ boolean
while (0) { }   // C.E: int ≠ boolean`} />

<Aside type="caution">
`"true"` (with quotes) is a **String**, not a boolean literal.  
`True` or `TRUE` → compile error. Java is case-sensitive for `true`/`false`.
</Aside>

---

## 6️⃣ null Literal

Represents the **absence of an object reference**.

<Code lang="java" title="null usage" code={`String s   = null;   // ✅ reference type
int[]  arr = null;   // ✅ array (reference type)

int     x  = null;   // ❌ C.E: primitive cannot be null
boolean b  = null;   // ❌ C.E: primitive cannot be null

Integer x  = null;   // ✅ wrapper class (object) can be null`} />

<Aside type="danger">
**Most common runtime crash in Java:**
```java
String s = null;
s.length();   // 💥 NullPointerException at runtime!
```
Always null-check before calling methods on a reference.
</Aside>

## 🔬 Literal Type Compatibility

1. **Check the literal type** — is it integer, float, char, etc.?
2. **Check the target variable type** — what is declared on the left?
3. **Check the range** — does the literal value fit?
4. **Assignment allowed** ✅ or **Compile Error** ❌


<Code lang="java" title="Compatibility examples" code={`byte b  = 10;    // ✅ 10 fits in byte range (−128 to 127)
byte b  = 130;   // ❌ C.E: 130 > 127 — possible loss of precision
int  i  = 10.5;  // ❌ C.E: can't assign double to int
long l  = 100;   // ✅ int literal auto-promoted to long
float f = 10.5;  // ❌ C.E: double literal needs 'f' suffix → 10.5f`} />

## 🔥 Interview Traps

<CardGrid>
  <Card title="Trap 1 — Missing suffix" icon="error">
    ```java
    float f = 10.5;           // ❌ (needs f)
    long  l = 9_000_000_000;  // ❌ (needs L)
    ```
  </Card>
  <Card title="Trap 2 — Octal gotcha" icon="error">
    ```java
    int x = 010;  // NOT 10 — it's OCTAL = 8!
    int x = 090;  // ❌ C.E: 9 invalid in octal
    ```
  </Card>
  <Card title="Trap 3 — char arithmetic" icon="error">
    ```java
    char c = 'A' + 1;       // ❌ result is int
    char c = (char)('A'+1); // ✅ cast needed
    ```
  </Card>
  <Card title="Trap 4 — null on primitives" icon="error">
    ```java
    int x = null;     // ❌ primitive ≠ null
    Integer x = null; // ✅ wrapper class = ok
    ```
  </Card>
</CardGrid>

<Aside type="tip">
**DSA connection — char literals:**  
`'a'` = 97, `'z'` = 122, `'A'` = 65, `'0'` = 48  
The pattern `char - 'a'` gives indices 0–25 — foundation of frequency arrays, sliding window, and Trie implementations.
</Aside>
