# 🔌 Interfaces in Java — Complete Guide

---

## 🗂️ Interface Declaration & Implementation

> An **Interface** is a 100% abstract contract — it defines **WHAT** to do, not **HOW**.

```java
// Declaration
interface Animal {
    void eat();    // method
    void sleep();  // method
}

// Implementation
class Dog implements Animal {
    @Override
    public void eat() {            // MUST implement all methods
        System.out.println("Dog eating");
    }
    @Override
    public void sleep() {
        System.out.println("Dog sleeping");
    }
}
```

```
Interface rules:
├── Cannot be instantiated (new Animal() ❌)
├── Class MUST implement ALL methods
│   OR class must be declared abstract
├── One class can implement MULTIPLE interfaces ✅
└── Reference variable CAN be of interface type ✅
    Animal a = new Dog(); // ✅
```
Note1:
Whenever we are implementing an interface compulsory for every method of that interface we should provide implementation otherwise we have to declare class as abstract in that case child class is responsible to provide implementation for remaining methods.
Note2: 
Whenever we are implementing an interface method compulsory it should be declared as public otherwise we will get compile time error.
Example: 
```java
interface Interf {
void methodOne();
void methodTwo();
}
abstrac class ServiceProvider implements Interf{ publiovoid methodOne(){}
}

class SubServiceProvider extends ServiceProvider{}
Output:
Compile time error.
D:\Java>javac SubServiceProvider.java
SubServiceProvider.java:1: 
SubServiceProvider is not abstract and does not override abstract method methodTwo() in Interf class SubServiceProvider extends ServiceProvider
```

## 🔄 extends vs implements

```
extends   → class to class     (single)
            interface to interface (multiple ✅)

implements → class to interface (multiple ✅)
```

```java
// Class extends Class (single only)
class Dog extends Animal { }         // ✅
class Dog extends Animal, Pet { }    // ❌ not allowed!

// Class implements Interface (multiple ✅)
class Dog implements Flyable, Swimmable, Runnable { } // ✅

// Interface extends Interface (multiple ✅)
interface C extends A, B { }         // ✅ interface CAN extend multiple!

// Class extends + implements both
class Dog extends Animal
          implements Flyable, Swimmable { } // ✅ valid!
// extends MUST come before implements ← order matters!
```

## 🛠️ Interface Methods

### Evolution of Interface Methods Across Java Versions

```
┌──────────────┬────────────────────────────────────────────┐
│ Java Version │ What's allowed in Interface                │
├──────────────┼────────────────────────────────────────────┤
│ Before Java 8│ abstract methods only                      │
│ Java 8       │ + default methods + static methods         │
│ Java 9+      │ + private methods + private static methods │
└──────────────┴────────────────────────────────────────────┘
```

### 1️⃣ Abstract Methods (Original)
```java
interface Shape {
    double area();      // implicitly public + abstract
    double perimeter(); // no body!
}
```

```
Interface method implicit modifiers:
Every interface method is automatically:
public + abstract (before Java 8)
You don't need to write them — compiler adds!

interface Shape {
    double area();
    // same as: public abstract double area();
}
```

### 2️⃣ Default Methods (Java 8+)
```java
interface Vehicle {
    void start();              // abstract — must implement

    default void fuelCheck() { // default — optional override ✅
        System.out.println("Checking fuel...");
    }
}

class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("Car started");
    }
    // fuelCheck() — no need to override, default used ✅
    // OR override it if you want custom behavior
}
```

```
Why default methods?
→ Before Java 8: adding new method to interface
  BREAKS all implementing classes (must implement!)
→ With default: add new method with default body
  existing classes NOT broken ✅
→ Used extensively in Collections API (Java 8 streams)
```

### 3️⃣ Static Methods (Java 8+)
```java
interface MathUtils {
    static int square(int n) {   // belongs to interface, not impl class
        return n * n;
    }
}

// Call via interface name only!
MathUtils.square(5);  // ✅
// Dog.square(5);     // ❌ NOT inherited by implementing class!
```

### 4️⃣ Private Methods (Java 9+)
```java
interface Logger {
    default void logInfo(String msg) {
        log("INFO", msg);      // reuse private helper
    }
    default void logError(String msg) {
        log("ERROR", msg);     // reuse private helper
    }
    private void log(String level, String msg) { // helper — not exposed!
        System.out.println("[" + level + "] " + msg);
    }
}
```

## 📦 Interface Variables

```java
interface Config {
    int MAX_SIZE = 100;           // interface variable
    String BASE_URL = "api.com";  // interface variable
}
```

```
Interface variable implicit modifiers:
Every variable is automatically:
public + static + final

int MAX_SIZE = 100;
// same as: public static final int MAX_SIZE = 100;

Rules:
├── MUST be initialized at declaration ✅ (final!)
├── Cannot be changed (final)
├── Shared across all (static)
├── Accessible everywhere (public)
└── No instance variables in interface!
```

```java
// ⚠️ Interview Trap
interface Demo {
    int x = 10;      // public static final — automatically!
}

class MyClass implements Demo {
    void show() {
        x = 20;       // ❌ COMPILE ERROR — it's final!
        System.out.println(x); // ✅ read is fine
    }
}
```

## ⚔️ Interface Naming Conflicts

### 🔹 Method Naming Conflicts

#### Case 1 — Same signature, same return type ✅
```java
interface A { void display(); }
interface B { void display(); }

// No conflict — one implementation satisfies both!
class C implements A, B {
    @Override
    public void display() {       // ✅ serves both A and B
        System.out.println("display");
    }
}
```

#### Case 2 — Same name, different return type ❌
```java
interface A { int display(); }
interface B { void display(); }

class C implements A, B {
    // ❌ COMPILE ERROR — impossible to satisfy both!
    // Can't have two methods with same name
    // different return types only (not valid overloading)
}
```

#### Case 3 — Default method conflict ⚠️
```java
interface A {
    default void show() { System.out.println("A"); }
}
interface B {
    default void show() { System.out.println("B"); }
}

class C implements A, B {
    // ❌ COMPILE ERROR if not resolved!
    // MUST override to resolve conflict:
    @Override
    public void show() {
        A.super.show(); // explicitly call A's default ✅
        // OR B.super.show();
        // OR own implementation
    }
}
```

### 🔹 Variable Naming Conflicts

```java
interface A { int x = 10; }
interface B { int x = 20; }

class C implements A, B {
    void show() {
        System.out.println(x);   // ❌ COMPILE ERROR — ambiguous!
        System.out.println(A.x); // ✅ resolve with interface name
        System.out.println(B.x); // ✅ resolve with interface name
    }
}
```

## 🏷️ Marker Interface

> An interface with **NO methods and NO variables** — just marks a class for special behavior.

```java
// Marker interface — completely empty!
interface Serializable { }   // java.io.Serializable
interface Cloneable { }      // java.lang.Cloneable
interface RandomAccess { }   // java.util.RandomAccess
```

```
How marker interfaces work:
JVM / Framework checks:
if (obj instanceof Serializable) {
    // ok to serialize!
}

You don't implement any method —
just TAGGING the class for JVM/framework!
```

```java
// Using marker interface
class User implements Serializable { // ✅ marked as serializable
    String name;
    int age;
    // ObjectOutputStream will now serialize this object
}

class Secret { // NOT Serializable
    String data;
    // ObjectOutputStream will REJECT this ❌
}
```

```
Modern alternative to Marker Interface:
Annotations (@Annotation) — Java 5+
@Entity, @Component, @Deprecated
Same marking purpose — more flexible!
```

## 🔌 Adapter Class

> An **Adapter class** provides **default (empty) implementations** of all interface methods — so you only override what you need.

```java
// Interface with many methods
interface WindowListener {
    void windowOpened();
    void windowClosed();
    void windowClosing();
    void windowIconified();
    void windowDeiconified();
    void windowActivated();
    void windowDeactivated();
}

// Adapter — implements all with empty body
abstract class WindowAdapter implements WindowListener {
    public void windowOpened() { }      // empty default
    public void windowClosed() { }      // empty default
    public void windowClosing() { }     // empty default
    public void windowIconified() { }   // empty default
    public void windowDeiconified() { } // empty default
    public void windowActivated() { }   // empty default
    public void windowDeactivated() { } // empty default
}

// You extend adapter — override ONLY what you need!
class MyWindow extends WindowAdapter {
    @Override
    public void windowClosing() {       // only this one needed!
        System.out.println("Closing window...");
    }
    // rest are empty — no need to implement!
}
```

```
Without Adapter:
class MyWindow implements WindowListener {
    // MUST implement ALL 7 methods even if need only 1 😩
}

With Adapter:
class MyWindow extends WindowAdapter {
    // Override ONLY what you need ✅ — clean!
}
```

## 🆚 Interface vs Abstract Class vs Concrete Class

```
┌──────────────────┬────────────────┬───────────────┬──────────────┐
│    Feature       │  Interface     │ Abstract Class│Concrete Class│
├──────────────────┼────────────────┼───────────────┼──────────────┤
│ Instantiate      │ ❌             │ ❌            │ ✅           │
│ Constructor      │ ❌             │ ✅            │ ✅           │
│ Abstract methods │ ✅ (implicit)  │ ✅ (explicit) │ ❌           │
│ Concrete methods │ ✅ (default/   │ ✅            │ ✅           │
│                  │  static/priv)  │               │              │
│ Variables        │ public static  │ Any type      │ Any type     │
│                  │ final only     │               │              │
│ Multiple inherit │ ✅ (implement  │ ❌ (extend    │ ❌           │
│                  │  multiple)     │  one only)    │              │
│ Access modifiers │ public only    │ Any           │ Any          │
│ extends          │ interface only │ class/abstract│ class        │
│ implements       │ N/A            │ interface ✅  │ interface ✅ │
└──────────────────┴────────────────┴───────────────┴──────────────┘
```

## 🆚 Interface vs Abstract Class — Deep Difference

```
┌──────────────────────┬─────────────────────────────────────────┐
│      Interface       │           Abstract Class                │
├──────────────────────┼─────────────────────────────────────────┤
│ CAN-DO relationship  │ IS-A relationship                       │
│ "implements"         │ "extends"                               │
│ Dog implements       │ Dog extends Animal                      │
│ Flyable              │                                         │
├──────────────────────┼─────────────────────────────────────────┤
│ Multiple allowed     │ Single inheritance only                 │
│ class Dog implements │ class Dog extends Animal                │
│ A, B, C { }         │ (can't extend two classes)              │
├──────────────────────┼─────────────────────────────────────────┤
│ Variables:           │ Variables: any type                     │
│ public static final  │ instance, static, final, etc.          │
│ only                 │                                         │
├──────────────────────┼─────────────────────────────────────────┤
│ No constructor       │ Has constructor                         │
│                      │ (called via super())                    │
├──────────────────────┼─────────────────────────────────────────┤
│ All methods public   │ Methods can be any access modifier      │
│ (implicit)           │                                         │
├──────────────────────┼─────────────────────────────────────────┤
│ When: unrelated      │ When: related classes share             │
│ classes need same    │ common base behavior                    │
│ capability           │                                         │
│ e.g. Bird, Airplane  │ e.g. Dog, Cat extend Animal             │
│ both implement       │                                         │
│ Flyable              │                                         │
└──────────────────────┴─────────────────────────────────────────┘
```

```java
// When to use Interface:
// UNRELATED classes — same capability
class Bird    implements Flyable { } // Bird can fly
class Airplane implements Flyable { } // Airplane can fly
class Superman implements Flyable { } // Superman can fly
// No IS-A between them — just CAN-DO ✅

// When to use Abstract Class:
// RELATED classes — shared base
abstract class Animal {
    String name;           // shared field
    void breathe() { }     // shared behavior
    abstract void sound(); // each has own sound
}
class Dog extends Animal { void sound() { System.out.println("Woof"); } }
class Cat extends Animal { void sound() { System.out.println("Meow"); } }
// All ARE Animals — IS-A ✅
```

## 🎯 Interview Cheat Sheet

```
Q: Can interface extend multiple interfaces?
A: YES ✅ — interface C extends A, B { } is valid!

Q: Can interface have constructor?
A: NO ❌ — interfaces cannot be instantiated.

Q: Are interface methods public by default?
A: YES — implicitly public abstract (before Java 8)
   Java 9 added private methods to interfaces.

Q: Can interface variables be changed?
A: NO ❌ — public static final — value fixed at declaration.

Q: What is default method? Why introduced?
A: Method with body in interface. Java 8+.
   Introduced to add new methods to existing interfaces
   WITHOUT breaking all implementing classes.

Q: Two interfaces same default method — what happens?
A: COMPILE ERROR in implementing class unless overridden.
   Use: InterfaceName.super.method() to resolve.

Q: Can abstract class implement interface partially?
A: YES ✅ — abstract class can leave some methods unimplemented.
   Concrete subclass must implement remaining.

Q: Marker interface vs Annotation?
A: Marker interface: empty interface (Serializable, Cloneable)
   Annotation: @interface (modern, more flexible, carries metadata)

Q: What is adapter class?
A: Abstract class implementing interface with empty methods.
   Subclass overrides only needed methods. ✅

Q: Interface variable default modifiers?
A: public + static + final (automatically!)

Q: Can interface be private or protected?
A: NO ❌ for top-level interface.
   Inner interface can be private/protected.

Q: Can we declare main() in interface?
A: YES ✅ (Java 8+ static method in interface)
   But practically not done.
```

## 🧩 DSA Angle

```java
// 1. Comparator — functional interface (Java 8)
// Sort by length then alphabetically
Comparator<String> comp = (a, b) -> {
    if (a.length() != b.length())
        return a.length() - b.length();
    return a.compareTo(b);
};
Arrays.sort(words, comp); // ✅

// 2. Comparable — natural ordering
class Interval implements Comparable<Interval> {
    int start, end;
    @Override
    public int compareTo(Interval other) {
        return this.start - other.start; // sort by start
    }
}
Arrays.sort(intervals); // uses compareTo ✅

// 3. Iterable interface — custom DS traversal
class MyList<T> implements Iterable<T> {
    @Override
    public Iterator<T> iterator() {
        return new Iterator<T>() {
            int idx = 0;
            public boolean hasNext() { return idx < size; }
            public T next() { return data[idx++]; }
        };
    }
}
// Now works with for-each loop!
for (T item : myList) { } // ✅

// 4. Functional interfaces in DSA
Predicate<Integer> isEven = n -> n % 2 == 0;
Function<Integer, Integer> square = n -> n * n;
list.stream()
    .filter(isEven)
    .map(square)
    .collect(Collectors.toList());
```

## 🏢 Real-Life SDE Usage

```java
// 1. Strategy Pattern using Interface
interface PaymentStrategy {
    void pay(double amount);
}

class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("Paid " + amount + " via Credit Card");
    }
}

class UPIPayment implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("Paid " + amount + " via UPI");
    }
}

class OrderService {
    private PaymentStrategy strategy; // program to interface!

    OrderService(PaymentStrategy strategy) {
        this.strategy = strategy;
    }

    void checkout(double amount) {
        strategy.pay(amount); // runtime polymorphism ✅
    }
}

// Usage — switch strategy at runtime!
new OrderService(new UPIPayment()).checkout(500);
new OrderService(new CreditCardPayment()).checkout(1000);

// 2. Repository Pattern (Spring Boot)
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);  // Spring auto-implements!
    List<User> findByStatus(String status);
}
// Interface → Spring generates implementation at runtime! 🔥

// 3. Marker Interface still used today
class SensitiveData implements Serializable, Cloneable {
    private transient String password; // transient + serializable ✅
}
```
