## 🔢 Bitwise Operators

```text
&    bitwise AND
|    bitwise OR
^    bitwise XOR (exclusive OR)
~    bitwise complement (NOT)
<<   left shift
>>   right shift (signed, preserves sign)
>>>  unsigned right shift (fills with 0)
```

<Aside type="note">
**Key Rule**: Bitwise operators work on **integral types only** (`byte`, `short`, `int`, `long`, `char`).  
❌ They **cannot** be applied to `boolean` for `~`, `<<`, `>>`, `>>>`.  
✅ However, `&`, `|`, `^` **do work on boolean** as logical operators (see below).
</Aside>

---

## 🔬 Boolean Bitwise Operators (`&`, `|`, `^`)

Unlike `&&` and `||`, the single-character `&` and `|` **do not short-circuit** — both operands are always evaluated.

<Code lang="java" title="Boolean bitwise truth tables" code={`// AND (&): both must be true
System.out.println(true & true);    // true
System.out.println(true & false);   // false
System.out.println(false & false);  // false

// OR (|): at least one true
System.out.println(true | false);   // true
System.out.println(false | false);  // false

// XOR (^): exactly one true (different)
System.out.println(true ^ false);   // true
System.out.println(true ^ true);    // falseSystem.out.println(false ^ false);  // false

// Non-short-circuit behavior:
boolean check() { System.out.println("evaluated"); return false; }
if (false & check()) { }  // "evaluated" prints! (check() runs even though false)
if (false && check()) { } // "evaluated" does NOT print (short-circuits)`} />

<Aside type="caution">
**When to use `&`/`|` on booleans**: Rarely. Prefer `&&`/`||` for efficiency. Use `&`/`|` only when you **need both sides evaluated** (e.g., for side effects).
</Aside>

## 🔢 Integral Bitwise Operations — Binary Examples

<Code lang="java" title="Bitwise AND, OR, XOR, NOT" code={`int a = 5;  // binary: 0101
int b = 3;  // binary: 0011

// AND (&): 1 only if both bits are 1
System.out.println(a & b);  // 0101 & 0011 = 0001 → 1

// OR (|): 1 if any bit is 1
System.out.println(a | b);  // 0101 | 0011 = 0111 → 7

// XOR (^): 1 if bits differ
System.out.println(a ^ b);  // 0101 ^ 0011 = 0110 → 6

// NOT (~): flip all bits (2's complement)
System.out.println(~a);     // ~0101 = ...11111010 → -6 (see 2's complement below)`} />

## 🔁 Bitwise Complement (`~`) & 2's Complement

### How `~` Works

The `~` operator flips **every bit**: `0 → 1`, `1 → 0`.

```text
a = 5 (int, 32-bit):
00000000 00000000 00000000 00000101  → 5

~a:
11111111 11111111 11111111 11111010  → ? (negative number)```

### 🧮 2's Complement: How Java Represents Negative Numbers

1. **Positive numbers**: Stored directly in binary.
2. **Negative numbers**: Stored as **2's complement**:
   - Step 1: Flip all bits (1's complement)
   - Step 2: Add 1

<Code lang="java" title="Why ~5 = -6" code={`// ~5 calculation:
// 5 in binary:     000...0101
// Flip bits:       111...1010  (1's complement)
// Add 1:           111...1011  (2's complement)

// To read 111...1011 as decimal:
// It's negative (MSB = 1), so:
// Step 1: Subtract 1 → 111...1010
// Step 2: Flip bits  → 000...0101 = 5
// Result: -5 - 1 = -6 ✅

System.out.println(~5);  // -6

// General formula: ~x = -(x + 1)
System.out.println(~0);   // -1
System.out.println(~10);  // -11
System.out.println(~(-1)); // 0`} />

<Aside type="tip">
**Memory Trick**: `~x = -(x + 1)`  
Use this to compute complement results instantly in interviews!
</Aside>

## 🔄 Shift Operators — Fast Multiply/Divide by Powers of 2

### Left Shift (`<<`)

```text
a << n  →  multiply a by 2^n  (fast!)
```

<Code lang="java" title="Left shift examples" code={`int a = 5;  // 0101

System.out.println(a << 1);  // 1010 = 10  (5 × 2¹)
System.out.println(a << 2);  // 010100 = 20 (5 × 2²)
System.out.println(a << 3);  // 0101000 = 40 (5 × 2³)

// Practical: Fast power of 2
int pow2 = 1 << 10;  // 1024 (2¹⁰) ✅`} />
### Signed Right Shift (`>>`)

```text
a >> n  →  divide a by 2^n (rounds toward negative infinity)
```

<Code lang="java" title="Signed right shift — preserves sign" code={`int pos = 20;  // 000...10100
System.out.println(pos >> 1);  // 10  (20 ÷ 2)
System.out.println(pos >> 2);  // 5   (20 ÷ 4)

int neg = -8;  // 111...11000 (2's complement)
System.out.println(neg >> 1);  // -4  (sign bit 1 is preserved!)
System.out.println(neg >> 2);  // -2

// Visual: -8 >> 1
// Before: 11111111 11111111 11111111 11111000
// After:  11111111 11111111 11111111 11111100  → -4 ✅`} />

### Unsigned Right Shift (`>>>`)

```text
a >>> n  →  shift right, always fill left with 0 (ignores sign)
```

<Code lang="java" title="Unsigned right shift — fills with 0" code={`int pos = 20;
System.out.println(pos >>> 1);  // 10  (same as >> for positive)

int neg = -8;
System.out.println(neg >> 1);   // -4  (signed: preserves sign bit)
System.out.println(neg >>> 1);  // 2147483644  (unsigned: fills with 0!)

// Visual: -8 >>> 1
// Before: 11111111 11111111 11111111 11111000
// After:  01111111 11111111 11111111 11111100  → large positive!
// = 2³¹ - 4 = 2147483644 ✅`} />

<Aside type="danger">
**Critical Difference**:  
- `>>` preserves sign → safe for signed arithmetic  
- `>>>` always fills with 0 → use for **bit masks**, **hashing**, or **unsigned logic**  
Never use `>>>` for signed division — it produces wrong results for negatives!
</Aside>

## ⚠️ Rules & Compile-Time Restrictions

<CardGrid>
  <Card title="Rule 1: Integral Types Only (Mostly)" icon="error">```java
    // ✅ Valid for byte, short, int, long, char:
    int x = 5;
    System.out.println(~x);      // -6
    System.out.println(x << 2);  // 20
    
    // ❌ Invalid for boolean (except &, |, ^):
    boolean b = true;
    System.out.println(~b);      // ❌ C.E: operator ~ cannot be applied to boolean
    System.out.println(b << 1);  // ❌ C.E: operator << cannot be applied to boolean
```
  </Card>
  
  <Card title="Rule 2: Type Promotion Applies" icon="information">
```java
    byte b = 5;
    // b << 1 → b is promoted to int, result is int
    int result = b << 1;  // ✅
    
    // Cannot assign back to byte without cast:
    // byte b2 = b << 1;  // ❌ C.E: possible loss of precision
    byte b2 = (byte)(b << 1);  // ✅ explicit cast
```
    **Why**: All bitwise operations on `byte`/`short`/`char` promote to `int` first.
  </Card>

  <Card title="Rule 3: Shift Distance Modulo 32 (for int)" icon="backspace">
```java
    int x = 1;
    System.out.println(x << 32);  // 1, NOT 0!
    // Shift distance is taken modulo 32 for int:
    // 32 % 32 = 0 → x << 0 = x
    
    // For long, modulo 64:
    long y = 1L;
    System.out.println(y << 64);  // 1L (64 % 64 = 0)
```
    **Interview Trap**: `x << 32` does **not** zero out an `int` — it's equivalent to `x << 0`.
  </Card>
</CardGrid>

## 🧩 DSA — Essential Bitwise Tricks

<CardGrid>
  <Card title="Trick 1: Even/Odd Check" icon="rocket">
```java
    // Check if n is even (last bit = 0)
    if ((n & 1) == 0) { /* even */ }    
    // Check if n is odd (last bit = 1)
    if ((n & 1) == 1) { /* odd */ }
    
    // Faster than n % 2 == 0!
```
  </Card>
  
  <Card title="Trick 2: Power of Two Check" icon="shield">
```java
    // n is power of 2 if: n > 0 AND (n & (n-1)) == 0
    boolean isPowerOfTwo(int n) {
        return n > 0 && (n & (n - 1)) == 0;
    }
    
    // Why? Powers of 2 have single 1-bit:
    // 8 = 1000, 7 = 0111 → 1000 & 0111 = 0000 ✅
    // 6 = 0110, 5 = 0101 → 0110 & 0101 = 0100 ≠ 0 ❌
```
  </Card>

  <Card title="Trick 3: Swap Without Temp Variable" icon="backspace">
```java
    // XOR swap (works for integers)
    a = a ^ b;
    b = a ^ b;  // (a^b)^b = a
    a = a ^ b;  // (a^b)^a = b
    
    // ⚠️ Warning: Don't use in production — less readable, 
    // and fails if a and b reference same memory location.
    // Prefer: int temp = a; a = b; b = temp;
```
  </Card>

  <Card title="Trick 4: Bit Manipulation Utilities" icon="information">
```java
    // Get i-th bit (0 = LSB)
    int getBit(int n, int i) {
        return (n >> i) & 1;
    }
    
    // Set i-th bit to 1
    int setBit(int n, int i) {
        return n | (1 << i);
    }
    
    // Clear i-th bit to 0
    int clearBit(int n, int i) {
        return n & ~(1 << i);
    }    
    // Toggle i-th bit
    int toggleBit(int n, int i) {
        return n ^ (1 << i);
    }
    
    // Check if i-th bit is set
    boolean isBitSet(int n, int i) {
        return ((n >> i) & 1) == 1;
    }
```
  </Card>
</CardGrid>

<Aside type="tip">
**DSA Pro Tip**: Bitwise operations are **O(1)** and extremely fast — ideal for:
- Frequency counting with bit masks (`int` for 32 flags, `long` for 64)
- Subset generation (`1 << n` subsets for n elements)
- XOR-based problems (find unique element, swap, encryption)
- Low-level optimization in competitive programming
</Aside>

## 🎯 Interview Cheat Sheet

<CardGrid>
  <Card title="Q: What is ~5?" icon="information">
    **`-6`** — use formula `~x = -(x + 1)`.  
    `~5 = -(5 + 1) = -6` ✅
  </Card>
  
  <Card title="Q: Difference between >> and >>>?" icon="error">
    - `>>` (signed): Preserves sign bit (fills with 1 for negatives)  
    - `>>>` (unsigned): Always fills with 0, ignoring sign  
    Example: `-8 >> 1 = -4`, but `-8 >>> 1 = 2147483644`
  </Card>

  <Card title="Q: What does `n & (n-1)` do?" icon="rocket">
    **Clears the lowest set bit** in `n`.  
    - If result is `0` and `n > 0` → `n` is a power of 2  
    - Used to count set bits: `while (n > 0) { count++; n &= (n-1); }`
  </Card>

  <Card title="Q: Can bitwise operators be used on boolean?" icon="caution">
    **Only `&`, `|`, `^`** work on `boolean` (as non-short-circuit logical operators).  
    `~`, `<<`, `>>`, `>>>` → ❌ Compile Error on `boolean`.
  </Card>

  <Card title="Q: Why is `(n & 1)` faster than `n % 2`?" icon="approve-check">    Bitwise AND is a **single CPU instruction**, while modulo involves division.  
    Modern JVMs may optimize `% 2` to `& 1`, but explicit `& 1` guarantees the optimization.
  </Card>

  <Card title="Q: What is `1 << 32` for an int?" icon="error">
    **`1`** — shift distance is taken **modulo 32** for `int`.  
    `32 % 32 = 0` → `1 << 0 = 1`. Common interview trap!
  </Card>
</CardGrid>

### 🧮 Formulas to Memorize

| Expression | Equivalent | Use Case |
|------------|------------|----------|
| `~x` | `-(x + 1)` | Compute complement instantly |
| `x << n` | `x × 2ⁿ` | Fast multiplication |
| `x >> n` | `x ÷ 2ⁿ` (floor for negative) | Fast signed division |
| `x & 1` | `x % 2` | Even/odd check |
| `x & (x-1)` | Clears lowest set bit | Power-of-2 check, bit counting |
| `x ^ x` | `0` | Cancel out duplicates (XOR trick) |

<Aside type="caution">
**Final Checklist**:
1. ✅ Bitwise ops work on `byte`/`short`/`int`/`long`/`char` — **not** `float`/`double`
2. ✅ `&`, `|`, `^` also work on `boolean` (non-short-circuit)3. ✅ `~x = -(x + 1)` — memorize this formula!
4. ✅ Shift distance modulo 32 for `int`, modulo 64 for `long`
5. ✅ `>>` preserves sign; `>>>` always fills with 0
6. ✅ Use `(n & (n-1)) == 0` to check power of 2 (with `n > 0`)
7. ✅ Prefer `&&`/`||` over `&`/`|` for boolean logic (short-circuit is safer)
</Aside>
