## 🔗 String Concatenation Operator `+`

> When `+` involves a **String** — it becomes **concatenation**, not arithmetic addition.

```java
// String + anything = String concatenation (non-String is converted via toString())
System.out.println("Hello" + " World");  // "Hello World"
System.out.println("Age: " + 25);        // "Age: 25"
System.out.println("Pi: " + 3.14);       // "Pi: 3.14"
System.out.println("Flag: " + true);     // "Flag: true"
System.out.println("Value: " + null);    // "Value: null"
```

<Aside type="note">
**Key Rule**: If **either operand** is a `String`, the `+` operator performs **string concatenation**. The non-String operand is automatically converted to a String using `String.valueOf()`.
</Aside>

## ⚠️ Left-to-Right Evaluation Trap

> Java evaluates expressions **strictly left-to-right**. The position of the first `String` determines whether `+` acts as **addition** or **concatenation**.

### 🔄 Evaluation Flow Diagram

```text
Expression: 1 + 2 + " Hello"
Step 1: 1 + 2 → both int → arithmetic → 3
Step 2: 3 + " Hello" → String present → concat → "3 Hello"
Result: "3 Hello" ✅

Expression: "Hello " + 1 + 2
Step 1: "Hello " + 1 → String present → concat → "Hello 1"
Step 2: "Hello 1" + 2 → String present → concat → "Hello 12"
Result: "Hello 12" ✅
```
### 💻 Code Examples

<Code lang="java" title="Left-to-right evaluation in action" code={`System.out.println(1 + 2 + " Hello");    // "3 Hello"  ← 1+2=3 first, then concat
System.out.println("Hello " + 1 + 2);      // "Hello 12" ← concat "Hello 1", then "Hello 12"
System.out.println("Hello " + (1 + 2));    // "Hello 3"  ← parentheses force arithmetic first

// More complex examples:
System.out.println(10 + 20 + " result");   // "30 result" (10+20=30, then concat)
System.out.println("result: " + 10 + 20);  // "result: 1020" (concat 10, then concat 20)
System.out.println("result: " + (10 + 20));// "result: 30" (parentheses save the day!)`} />

### 📊 Comprehensive Examples Table

| Expression | Evaluation Steps | Result | Why? |
|------------|-----------------|--------|------|
| `"ashok" + 10 + 20 + 30` | `"ashok"+10` → `"ashok10"` → `+20` → `"ashok1020"` → `+30` | `"ashok102030"` | First operand is String → all subsequent `+` are concat |
| `10 + 20 + 30 + "ashok"` | `10+20=30` → `30+30=60` → `60+"ashok"` | `"60ashok"` | Arithmetic until String appears, then concat |

<Aside type="tip">
**Memory Trick**: *"Once a String enters the party, everything after becomes a String!"*  
The moment the left operand is a String (or becomes one via concat), all remaining `+` operations switch to concatenation mode.
</Aside>

## 🔬 char + char Trap

> `char + char` performs **arithmetic addition** (Unicode values), **not** string concatenation!

### Why This Happens

- `char` is a **numeric primitive** (2-byte Unicode value)
- Per arithmetic promotion rules: `char + char → int`
- Result is the **sum of Unicode code points**, not a concatenated string

<Code lang="java" title="char arithmetic vs string concatenation" code={`char a = 'A', b = 'B';

// ❌ Trap: char + char = int (Unicode addition)
System.out.println(a + b);           // 131 (65 + 66) — NOT "AB"!System.out.println((char)(a + b));   // 'ƃ' — Unicode character 131

// ✅ Fix: Force String context for concatenation
System.out.println("" + a + b);      // "AB" ✅ — empty String triggers concat mode
System.out.println(a + "" + b);      // "AB" ✅ — same effect
System.out.println(String.valueOf(a) + b);  // "AB" ✅ — explicit conversion

// ✅ Alternative: Use StringBuilder for multiple chars
System.out.println(new StringBuilder().append(a).append(b));  // "AB"`} />

### 📊 char Operation Reference

| Expression | Type Promotion | Result | Explanation |
|------------|---------------|--------|-------------|
| `'A' + 'B'` | `char + char → int` | `131` | Unicode: 65 + 66 |
| `'a' + 1` | `char + int → int` | `98` | 'a'=97, so 97+1=98 ('b') |
| `'a' + 1.0` | `char + double → double` | `98.0` | Promoted to double |
| `"" + 'A' + 'B'` | `String + char → String` | `"AB"` | Empty String triggers concat |
| `'A' + "" + 'B'` | `char + String → String` | `"AB"` | String appears early |

<Aside type="caution">
**DSA Connection**: This behavior is actually **useful** for algorithms!  
`char - 'a'` gives index 0–25 for lowercase letters — foundation of frequency arrays, sliding window, and Trie implementations.
```java
char c = 'd';
int index = c - 'a';  // 100 - 97 = 3 ✅
```
</Aside>

  <Card title="Q: What is `'A' + 'B'`?" icon="error">
    **`131`** (an `int`, not a String!)  
    `'A'`=65, `'B'`=66 → 65+66=131.  
    To get `"AB"`, use `"" + 'A' + 'B'` or `String.valueOf('A') + 'B'`.
  </Card>
  <Card title="Q: How to force arithmetic when String is present?" icon="approve-check">
    **Use parentheses** to evaluate arithmetic first:
```java
    int x = 10, y = 20;
    System.out.println("Sum: " + (x + y));  // "Sum: 30" ✅
    // Without parentheses: "Sum: " + x + y → "Sum: 1020" ❌
```
  </Card>

  <Card title="Q: What is the type of `"text" + 5`?" icon="information">
    **`String`** — whenever one operand is `String`, the result of `+` is always `String`.
  </Card>

  <Card title="Q: Does `"null" + null` throw NullPointerException?" icon="approve-check">
    **No!** → `"nullnull"`  
    `String.valueOf(null)` returns the literal string `"null"`, not a NPE.
```java
    System.out.println("Value: " + null);  // "Value: null" ✅
```
  </Card>

## 🧩 DSA & Practical Patterns

<CardGrid>
  <Card title="Pattern: Building Output Strings" icon="rocket">
    For simple cases, `+` concatenation is readable:
```java
    String name = "Alice";
    int age = 30;
    System.out.println(name + " is " + age + " years old");
    // "Alice is 30 years old" ✅
```
    **But**: For loops or many concatenations, use `StringBuilder` for performance:
```java
    // ❌ Inefficient in loop (creates many temporary Strings):
    String result = "";
    for (int i = 0; i < n; i++) {
        result += i + ",";  // O(n²) time!
    }
    
    // ✅ Efficient:
    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < n; i++) {
        sb.append(i).append(",");  // O(n) time
    }
    String result = sb.toString();
```  </Card>
  
  <Card title="Pattern: Debugging Expressions" icon="shield">
    When debugging complex `+` expressions, add explicit parentheses or intermediate variables:
```java
    // Hard to read:
    System.out.println("Result: " + a + b * c + d);
    
    // Clear and intentional:
    int product = b * c;
    int sum = a + product + d;
    System.out.println("Result: " + sum);
```
  </Card>

  <Card title="Pattern: Type-Safe Formatting" icon="backspace">
    Prefer `String.format()` or `System.out.printf()` for complex output:
```java
    // ❌ Error-prone concatenation:
    System.out.println("User: " + name + ", Score: " + score + "/" + max);
    
    // ✅ Clear and locale-aware:
    System.out.printf("User: %s, Score: %d/%d%n", name, score, max);
    // Or:
    String msg = String.format("User: %s, Score: %d/%d", name, score, max);
```
  </Card>
</CardGrid>

<Aside type="tip">
**Pro Checklist**:
1. ✅ Remember: evaluation is **strictly left-to-right**
2. ✅ First `String` operand switches all subsequent `+` to concatenation
3. ✅ Use **parentheses** to force arithmetic order: `"Result: " + (a + b)`
4. ✅ `char + char` → `int` (Unicode math), not String concat
5. ✅ For performance: use `StringBuilder` in loops, `String.format()` for complex output
6. ✅ `null + String` → `"null"` (no NPE) — `String.valueOf(null)` handles it safely
</Aside>
