## 2️⃣ Arithmetic Operators

```text
+   addition
-   subtraction
*   multiplication
/   division
%   modulus (remainder)
```

<Code lang="java" title="Basic usage" code={`int a = 10, b = 3;

System.out.println(a + b);  // 13
System.out.println(a - b);  // 7
System.out.println(a * b);  // 30
System.out.println(a / b);  // 3  ← integer division! (not 3.33)
System.out.println(a % b);  // 1  ← remainder`} />

## 🔢 Rule 1: Type Promotion in Arithmetic

> When applying any arithmetic operation between two operands `a` and `b`, the result type is always:
> ```
> max(int, type_of_a, type_of_b)
> ```

This means **smaller types are promoted to `int` or larger** before the operation.

### 📊 Promotion Examples

| Operand Types | Result Type | Example |
|--------------|-------------|---------|
| `byte + byte` | `int` | `byte a=1, b=2; var c = a+b; // c is int` |
| `byte + short` | `int` | `byte a=1; short s=2; var r = a+s; // int` |
| `short + short` | `int` | `short x=1, y=2; var z = x+y; // int` |
| `short + long` | `long` | `short s=1; long l=2L; var r = s+l; // long` |
| `float + double` | `double` | `float f=1.0f; double d=2.0; var r = f+d; // double` |
| `int + double` | `double` | `int i=1; double d=2.0; var r = i+d; // double` |
| `char + char` | `int` | `char c1='a', c2='b'; var r = c1+c2; // int (195)` |
| `char + int` | `int` | `char c='a'; int i=1; var r = c+i; // int (98)` |
| `char + double` | `double` | `char c='a'; double d=1.2; var r = c+d; // double (98.2)` |

### 💻 Character Arithmetic Examples

<Code lang="java" title="char participates as Unicode value" code={`System.out.println('a' + 'b');      // 195 (97 + 98)
System.out.println('a' + 1);         // 98  (97 + 1)
System.out.println('a' + 1.2);       // 98.2 (97.0 + 1.2)

// To get concatenated string, ensure one operand is String:
System.out.println("" + 'a' + 'b');  // "ab"
System.out.println('a' + "" + 'b');  // "ab"`} />

<Aside type="caution">
**Assignment Trap**: Because `byte + byte → int`, you cannot assign the result back to `byte` without explicit cast:
```java
byte a = 10, b = 20;
byte c = a + b;  // ❌ C.E: possible loss of precision (int → byte)
byte c = (byte)(a + b);  // ✅ Explicit cast required
```
</Aside>

## 🔥 Rule 2: Division by Zero — Integral vs Floating Point

### Integral Arithmetic (`byte`, `short`, `int`, `long`)

> No representation for infinity → **`ArithmeticException` at runtime**.

<Code lang="java" title="Integral division by zero" code={`System.out.println(10 / 0);    // 💥 Runtime: ArithmeticException: / by zero
System.out.println(10 % 0);    // 💥 Runtime: ArithmeticException: / by zero`} />

### Floating Point Arithmetic (`float`, `double`)

> IEEE 754 standard defines **infinity** and **NaN** → **No exception**, special values returned.

<Code lang="java" title="Floating point division by zero" code={`System.out.println(10 / 0.0);    // Infinity ✅
System.out.println(-10 / 0.0);   // -Infinity ✅
System.out.println(0.0 / 0.0);   // NaN ✅
System.out.println(-0.0 / 0.0);  // NaN ✅

// Access via wrapper class constants:
System.out.println(Float.POSITIVE_INFINITY);   // Infinity
System.out.println(Double.NEGATIVE_INFINITY);  // -Infinity
System.out.println(Float.NaN);                 // NaN`} />

<Aside type="note">
**Key Difference**:  
- Integral: `10/0` → 💥 `ArithmeticException`  
- Floating: `10.0/0.0` → `Infinity` (valid value, no crash)
</Aside>

## 🔬 Rule 3: NaN (Not a Number) Semantics

### Integral Arithmetic

> No representation for undefined results → **`ArithmeticException`**.

<Code lang="java" title="Integral undefined operations" code={`System.out.println(0 / 0);  // 💥 Runtime: ArithmeticException: / by zero`} />

### Floating Point Arithmetic

> Undefined results return **`NaN`** (Not a Number) — no exception.

<Code lang="java" title="Floating point NaN examples" code={`System.out.println(0.0 / 0.0);        // NaN ✅
System.out.println(Math.sqrt(-1));    // NaN ✅
System.out.println(Infinity - Infinity); // NaN ✅`} />

### ⚠️ NaN Comparison Rules — Critical Interview Trap

> **For ANY value `x` (including `NaN` itself), comparisons with `NaN` behave unexpectedly:**

<table>
  <thead>
    <tr><th>Expression</th><th>Result</th><th>Why?</th></tr>
  </thead>
  <tbody>
    <tr><td><code>10 &lt; Float.NaN</code></td><td><code>false</code></td><td>NaN is unordered</td></tr>
    <tr><td><code>10 &lt;= Float.NaN</code></td><td><code>false</code></td><td>NaN is unordered</td></tr>
    <tr><td><code>10 &gt; Float.NaN</code></td><td><code>false</code></td><td>NaN is unordered</td></tr>
    <tr><td><code>10 &gt;= Float.NaN</code></td><td><code>false</code></td><td>NaN is unordered</td></tr>
    <tr><td><code>10 == Float.NaN</code></td><td><code>false</code></td><td>NaN ≠ anything, even itself</td></tr>
    <tr><td><code>Float.NaN == Float.NaN</code></td><td><code>false</code></td><td>NaN is not equal to itself!</td></tr>
    <tr><td><code>10 != Float.NaN</code></td><td><code>true</code></td><td>Only != returns true with NaN</td></tr>
    <tr><td><code>Float.NaN != Float.NaN</code></td><td><code>true</code></td><td>NaN is not equal to itself!</td></tr>
  </tbody>
</table>

<Code lang="java" title="✅ Safe NaN checking" code={`double result = 0.0 / 0.0;

// ❌ Wrong way:
if (result == Double.NaN) { }  // Always false!

// ✅ Correct ways:
if (Double.isNaN(result)) { }  // Use library method
// OR
if (result != result) { }      // NaN is the only value != itself`} />

<Aside type="danger">
**Interview Gold**: `NaN == NaN` is **false** in Java (and IEEE 754). Always use `Double.isNaN(x)` or `Float.isNaN(x)` to check for NaN.
</Aside>

## ⚡ Rule 4: ArithmeticException Details

<CardGrid>
  <Card title="Key Facts" icon="information">
    1. ✅ It is a **`RuntimeException`** (unchecked) — not a compile-time error.
    2. ✅ Occurs **only in integral arithmetic** (`byte`, `short`, `int`, `long`).
    3. ✅ **Never occurs in floating point** (`float`, `double`) — they return `Infinity`/`NaN`.
    4. ✅ Only triggered by **`/` (division)** and **`%` (modulus)** operators.
    5. ✅ `+`, `-`, `*` never throw `ArithmeticException` (they may overflow silently).
  </Card>
  
  <Card title="Examples" icon="error">
    ```java
    // 💥 These throw ArithmeticException:
    int a = 10 / 0;    // / by zero
    int b = 10 % 0;    // / by zero
    
    // ✅ These do NOT throw (floating point):
    double c = 10.0 / 0.0;  // Infinity
    double d = 10.0 % 0.0;  // NaN
    
    // ✅ These do NOT throw (other operators):
    int max = Integer.MAX_VALUE;
    int overflow = max + 1;  // -2147483648 (silent wrap, no exception)
    ```
  </Card>
</CardGrid>

## 🎯 Common Traps & Solutions

<CardGrid>
  <Card title="Trap 1: Integer Division" icon="error">
    ```java
    int a = 7, b = 2;
    System.out.println(a / b);  // 3  NOT 3.5! 😱
    
    // ✅ Fix: Cast at least one operand to floating point
    System.out.println((double) a / b);  // 3.5 ✅
    System.out.println(a / (double) b);  // 3.5 ✅
    System.out.println(1.0 * a / b);     // 3.5 ✅
    ```
    **Rule**: If either operand is `float`/`double`, result is floating point.
  </Card>
  
  <Card title="Trap 2: Modulus Sign Rule" icon="error">
    ```java
    System.out.println( 10 %  3);  //  1
    System.out.println(-10 %  3);  // -1  ← sign follows DIVIDEND (left operand)!
    System.out.println( 10 % -3);  //  1
    System.out.println(-10 % -3);  // -1
    
    // ✅ Memory rule: result sign = sign of LEFT operand
    ```
    **Interview Answer**: Java's `%` is **remainder**, not mathematical modulo. Sign matches dividend.
  </Card>

  <Card title="Trap 3: Silent Integer Overflow" icon="error">
    ```java
    int max = Integer.MAX_VALUE;  // 2147483647
    System.out.println(max + 1);  // -2147483648 😱 wraps around silently!
    
    // ✅ Fix: Use long for large calculations
    long safe = (long) max + 1;   // 2147483648 ✅
    
    // ✅ Or use Math.addExact() to throw on overflow (Java 8+)
    try {
        int result = Math.addExact(max, 1);  // 💥 ArithmeticException
    } catch (ArithmeticException e) {
        System.out.println("Overflow detected!");
    }
    ```
  </Card>
</CardGrid>

## 🧩 DSA Angle: Arithmetic in Algorithms

<CardGrid>
  <Card title="Pattern: Safe Midpoint Calculation" icon="shield">
    Avoid overflow in binary search:
    ```java
    // ❌ Dangerous — can overflow:
    int mid = (low + high) / 2;
    
    // ✅ Safe — no overflow risk:
    int mid = low + (high - low) / 2;
    
    // ✅ Also safe (Java 8+):
    int mid = (low + high) >>> 1;  // unsigned right shift
    ```
  </Card>
  
  <Card title="Pattern: Modulus for Cyclic Indexing" icon="rocket">
    Use `%` for circular buffer/array access:
    ```java
    // Rotate array index safely:
    int next = (current + 1) % arr.length;  // wraps to 0 at end
    int prev = (current - 1 + arr.length) % arr.length;  // handles negative
    
    // Hash table indexing:
    int index = Math.abs(key.hashCode()) % tableSize;
    ```
  </Card>

  <Card title="Pattern: Detect Overflow in Competitive Coding" icon="backspace">
    When problem constraints suggest large numbers:
    ```java
    // If sum of n numbers (each ≤ 10⁹, n ≤ 10⁵) is needed:
    // Max sum = 10⁹ × 10⁵ = 10¹⁴ → exceeds int (2×10⁹)
    
    // ✅ Always use long for accumulators:
    long sum = 0;
    for (int val : arr) {
        sum += val;  // safe up to ~9×10¹⁸
    }
    ```
  </Card>
</CardGrid>

<Aside type="tip">
**Pro Checklist for Arithmetic**:
1. ✅ Remember: `byte/short/char` operations promote to `int`
2. ✅ Cast to `double` before division if you need decimal results
3. ✅ Use `Double.isNaN(x)` — never `x == NaN`
4. ✅ Watch for overflow: use `long` or `Math.addExact()` for safety
5. ✅ Modulus sign = sign of left operand (dividend)
6. ✅ Floating point never throws `ArithmeticException` — returns special values instead
</Aside>
