## What is an Identifier?

**Definition:** A name given to any component in a Java program.

| Used For   | Example                      | Identifier      |
|------------|------------------------------|-----------------|
| Variables  | `int age = 25;`              | `age`           |
| Methods    | `void calculateSum()`        | `calculateSum`  |
| Classes    | `class BankAccount`          | `BankAccount`   |
| Interfaces | `interface Flyable`          | `Flyable`       |
| Packages   | `com.company.utils`          | each part       |
| Constants  | `MAX_SIZE = 100`             | `MAX_SIZE`      |

### Example Breakdown

```java
class Test {
    public static void main(String[] args) {
        int x = 10;
    }
}
```

| Part     | Type               |
|----------|--------------------|
| `Test`   | Class              |
| `main`   | Method             |
| `String` | Predefined Class   |
| `args`   | Parameter          |
| `x`      | Variable           |

<Aside type="tip" title="Remember">
**Identifier = Just a NAME**  
But the compiler treats it as a **MEMORY REFERENCE**.
</Aside>

## The 6 Golden Rules ⭐

### Rule 1 — Allowed Characters

Only these 5 types are allowed:

1. `a` to `z`
2. `A` to `Z`
3. `0` to `9`
4. `_` (Underscore)
5. `$` (Dollar Sign)

<Aside type="caution">
Using anything else (`#`, `@`, `%`) → **Compile Time Error**
</Aside>

```java
✅ total_number
❌ Total#         // C.E.
```

---

### Rule 2 — Starting Character

Cannot start with a digit (`0–9`).

```java
✅ ABC123   // Valid → treated as identifier
❌ 123ABC   // Invalid → treated as a number
```

---

### Rule 3 — Case Sensitivity

Java is **case sensitive**.

```java
abc ≠ ABC ≠ Abc

int number = 10;   // different from →
int Number = 20;
```

---

### Rule 4 — Length Limit

Technically **no limit**, but keep it readable.

```java
numberOfStudents                            // ✅ Good
n                                           // ❌ Too vague
numberOfStudentsInEngineeringDepartment2024 // ❌ Too long
```

<Aside type="tip">
Best practice: keep identifiers **under 15 characters**.
</Aside>

---

### Rule 5 — Reserved Words

Cannot use Java **keywords** as identifiers.

```java
❌ int if = 10;  // invalid
```

---

### Rule 6 — Predefined Class Names

You **can** use predefined class names as identifiers — but avoid it.

```java
✅ int String = 10;  // compiles, outputs 10
⚠️  Legal but bad practice — very confusing
```

## Special Characters Deep Dive

### `$` Dollar Sign

✅ Allowed in identifiers.

```java
int $price = 100;
int total$amount = 500;
```

<Aside type="caution">
Technically valid but **not recommended** for normal code.  
`$` is mainly used by the **compiler** for generated classes.  
Example: `Outer$Inner.class` (inner class bytecode naming)
</Aside>

---

### `_` Underscore

<Aside type="danger">
Since **Java 9**, `_` alone is **not allowed** as an identifier.
</Aside>

```java
❌ int _ = 10;      // ERROR — reserved for future use

✅ int _value = 10;
✅ int total_sum = 50;
```

## Naming Conventions

> Not enforced by the compiler — but **mandatory in real-world SDE work**.

| What        | Convention        | Example                          |
|-------------|-------------------|----------------------------------|
| Variables   | `camelCase`       | `studentAge`, `maxScore`         |
| Methods     | `camelCase`       | `calculateTotal()`, `getUserById()` |
| Classes     | `PascalCase`      | `BankAccount`, `UserService`     |
| Interfaces  | `PascalCase`      | `Comparable`, `Runnable`         |
| Constants   | `UPPER_SNAKE_CASE`| `MAX_SIZE`, `PI`, `BASE_URL`     |
| Packages    | all lowercase     | `com.google.auth`, `org.spring`  |

## 🔬 Under the Hood — What Actually Happens?

When you write:

```java
int studentAge = 25;
```

The compiler stores `studentAge` in the **Symbol Table** during compilation.

After compilation, the `.class` (bytecode) file uses **memory addresses** — your identifier name is gone at runtime (unless reflection is used).

```
Source Code          →   Bytecode (.class)
int studentAge;      →   istore_1   ← just a memory slot number!
```

<Aside type="note">
Identifier names survive in `.class` files only as **debug metadata**.  
At JVM execution time, it's all **stack indices and memory addresses**.
</Aside>
